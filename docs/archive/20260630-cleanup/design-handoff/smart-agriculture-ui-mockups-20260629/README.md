# 智慧农业环境监测平台 UI 稿交接说明

## 目标

本包用于网页端统一处理页面视觉和信息架构。当前项目应做成一个自建的智慧农业主平台，OneNET 只作为旁路上报和云端可视化证明，不替代本系统。

核心定位：

- 主平台负责实时监测、历史分析、阈值控制、模型推算、告警日志和执行器控制。
- OneNET 负责旁路上报完整指标，用于实验 11/12 的云端展示与截图证明。
- 室外气象数据来自 Open-Meteo/高德等接口，只作为参考，不作为棚内实测来源。
- 板端真实采集字段与后端模型推算字段必须在 UI 上明确区分。

## 侧边栏规划

最终侧边栏控制在 6 项以内：

1. 实时监测
2. 历史分析
3. 控制与阈值
4. 模型管理
5. 环境气象
6. 告警日志

不要把 `设备管理` 和 `系统设置` 单独作为主导航：

- 设备管理并入 `控制与阈值` 页的 `设备映射` 区。
- 系统设置放入右上角 `管理员` 下拉菜单，避免主导航膨胀。

## 数据来源规则

所有关键指标都要有来源标签。建议标签如下：

| 来源 | 展示标签 | 用途 |
| --- | --- | --- |
| 板端真实采集 | 实测 | DHT11 温度、DHT11 空气湿度、P0.7 光敏光照 |
| 后端模型推算 | 模型 | 土壤湿度、CO2、土壤 EC、土壤 TDS、土壤肥力、红外状态 |
| 外部天气接口 | 气象接口 | 室外温度、降水、风速、辐射、AQI 等 |
| 控制/系统事件 | 控制 / 系统 | 水泵自动开启、串口网关断开、OneNET 上报状态 |

当前硬件口径：

- 温度：实测，DHT11 P2.0。
- 空气湿度：实测，DHT11 P2.0。
- 光照：目标为实测，P0.7 光敏 ADC。若 ADC 未完成，临时显示为 `板端模拟`，不要写成 BH1750。
- 土壤湿度：模型，由温度、空气湿度、光照、水泵状态和时间步长推算。
- CO2：模型，由光照、温度、红外事件、时间段和自然通风系数推算。
- 土壤 EC/TDS/肥力：模型，内部以 EC 为主状态，TDS 由 EC 换算，肥力指数由 EC 映射。
- 红外状态：模型，按 PIR 人体/活动事件模拟，0/1 状态。

## 页面说明

### 01 实时监测

实时监测页是主入口。页面需要按来源分区：

- `板端实测`：温度、空气湿度、光照。
- `模型推算`：土壤湿度、CO2、土壤 EC、土壤 TDS、土壤肥力、红外状态。
- `趋势分组视图`：不要把所有量纲混在一张图里，至少按实测趋势、土壤模型趋势、CO2/红外事件分组。
- `执行器控制`：水泵、施肥泵、灭虫灯。
- `最近告警`：每条告警显示来源标签。
- `室外气象参考`：明确仅参考，不作为棚内实测来源。

注意：截图中的侧边栏仍有旧项，最终实现请按本说明压缩为 6 项。

### 02 历史分析

历史页用于查看趋势和导出数据。需要按来源分组：

- 板端实测趋势：温度、空气湿度、光照。
- 模型推算趋势：土壤湿度、CO2、EC、TDS、土壤肥力、红外。
- 表格中保留来源列，至少显示 `实测 / 模型`。

修正点：

- 图稿里的 `模型字段 5 个` 应改为 `模型字段 6 个`。

### 03 控制与阈值

此页合并阈值配置、执行器策略和设备映射。

阈值分组：

- 实测阈值：温度、空气湿度、光照。
- 模型阈值：土壤湿度、CO2、土壤 EC、土壤肥力、红外触发策略。

执行器策略：

- 水泵：自动依据 `土壤湿度 · 模型`。
- 施肥泵：手动优先，建议依据 `土壤肥力 / EC · 模型`。
- 灭虫灯：手动控制，红外触发只产生告警或建议。

设备映射建议：

| 设备 | 板端引脚 | 说明 |
| --- | --- | --- |
| 水泵继电器 | P0.6 | 控制水泵继电器 |
| 水泵指示灯 | P1.0 | 水泵状态指示 |
| 施肥泵指示 | P1.1 | 当前用 LED 模拟 |
| 灭虫灯输出/指示 | P1.6 | 当前用 LED 模拟 |

命令名以实际固件为准，建议统一为：

- 水泵：`BLEGLED1` / `BLEKLED1`
- 施肥泵：`BLEGLED2` / `BLEKLED2`
- 灭虫灯：`BLEGLED3` / `BLEKLED3`

### 04 模型管理

模型页用于解释和管理缺失传感器的后端仿真模型。重点是让老师能看懂模型不是随机数。

需要包含：

- 土壤水量平衡模型。
- CO2 质量守恒模型。
- EC/TDS/肥力模型。
- 红外事件模型。
- 来源映射表。
- 模型输出预览。

修正点：

- 红外当前应标为 `模型 · PIR事件`，不要标为实测。

### 05 环境气象

此页只展示室外气象参考。

必须保留使用边界：

- 不替代板端传感器。
- 不直接控制水泵。
- 可用于报告对比和初始化参考。
- 断网不影响主平台控制。

室外土壤含水量如果展示，只能作为参考，不驱动棚内土壤模型。

### 06 告警日志

告警页用于证明系统判断和控制动作可追溯。

每条告警至少包含：

- 时间
- 级别
- 指标
- 当前值
- 阈值
- 来源
- 处理状态
- 触发动作

右侧详情需要展示决策链：

`输入数据 -> 阈值规则 -> 动作指令 -> 当前状态`

修正点：

- 图稿里左侧选中行和右侧详情有一处不一致，实际实现时必须保持一致。

## 实现注意事项

1. 不要再把所有数据称为 `实测`。只有板端真实采集才叫实测。
2. `模型` 不是随机数，应由真实温湿度、光照和执行器状态驱动。
3. `环境气象` 是外部接口，不是板端采集。
4. OneNET 是旁路上报，主平台不依赖 OneNET 才能运行。
5. 页面上所有标签和告警都要能回答老师追问：这个数据从哪里来。
6. 图表避免混合量纲误读，优先分组展示。
7. 右上角时间使用当前系统时间或最新数据时间，不能写死图稿里的日期。

## 推荐产品结构

系统主链路：

```text
CC2530终端 -> ZigBee协调器 -> 串口网关 -> FastAPI后端 -> Vue前端
```

后端补齐链路：

```text
实测温度/湿度/光照 + 执行器状态 -> 后端模型 -> 土壤/CO2/EC/TDS/肥力/红外
```

OneNET 旁路：

```text
FastAPI后端 -> OneNET物模型 -> OneNET控制台/大屏截图
```
