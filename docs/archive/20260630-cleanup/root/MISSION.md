# Mission: Smart Agriculture Defense Understanding

## Why
Build a clear, defensible understanding of the smart agriculture monitor so the team can explain the backend, hardware, serial bridge, interfaces, and experiment foundations during course defense.

## Success looks like
- Explain the full data path from CC2530 terminal node to Vue/FastAPI and back to actuator control.
- Distinguish real hardware measurements from simulated or pending fields without overstating the project.
- Map the current project to the relevant course experiments and supporting materials.
- Answer common defense questions with evidence from the repo and experiment folder.

## Constraints
- Use the current workspace and `/Users/admin/Downloads/实验课/` as the source of truth.
- Keep lessons short enough for quick review before defense.
- User-facing teaching materials should be in Chinese.

## Out of scope
- Rewriting firmware, backend, or frontend code.
- Claiming unverified original experiment source exists when only project docs or compressed materials were found.
