第 1 课：先认清系统里的角色

小白不要先背技术名词。先把系统想成一条“从地里采数据，到网页显示，再回去控制设备”的流水线。

1 角色 2 硬件 3 上行 4 下行 5 实验 术语表

你先记住一句话

终端板采集数据，协调器把无线数据转成串口，Python bridge 把串口转成网络消息，FastAPI 后端存库和判断，Vue 前端展示和发控制。

这句话就是答辩的骨架。后面所有细节都挂在这条骨架上。

每个角色到底干什么

终端 CC2530在大棚现场，读取 DHT11 温湿度，并接收命令控制继电器/LED。它是“干活的板”。
协调器 CC2530不负责展示页面，也不负责数据库。它只把 ZigBee 无线数据转成 UART 串口数据，是“中转站”。
Python bridgePC 上跑的小程序。它读串口，把 t:25-h:60 这种硬件字符串翻译成 WebSocket JSON。
FastAPI 后端系统大脑。数据入库、阈值判断、报警记录、自动控制命令都在这里完成。
Vue 前端用户看的页面。显示实时数据、历史曲线、报警日志，也提供手动控制按钮。

新手最容易混的点

不要把 ZigBee、串口、WebSocket 当成同一个东西。它们是不同路段：

ZigBee 是两块 CC2530 之间的无线；UART 是协调器到 PC；WebSocket 是 PC/后端/前端之间的网络实时通信。
